Statistical Graphics:

Data presented in a graphical format. Well-designed statistical graphic, also referred to as a chart or graph, consists of complex ideas communicated with clarity, precision, and efficiency.

Components of a Statistical Graphic:

  • Axes: Typically two, horizontal (X) and vertical (Y).
  • Data Presentation: Emphasize data, minimize non-data elements, and avoid unnecessary embellishment.
  • Axes Values: Increase as they move away from the origin.
  • Scales and Scaling: Use linear scales, start numeric scale at zero, and provide aids for scale interpretation.
  • Proportion: Ensure accurate proportion of displayed surfaces to the data they represent.
  • Lines: Data lines should be heavier than axes lines, and grid lines should be thin or absent.
  • Labeling: Detailed labeling integrated with the drawing; provide information about data.
  • Title: Create a short, clear title describing the purpose of the graphic.

Aiding Interpretation of Numbers:

Display a grid on request.

Allow viewers to click on a data point for actual values.

Show numeric values automatically for each point or bar.

Allow users to change scale values.

Types of Statistical Graphics:

  • Curve and Line Graphs: Limit to four or five curves, highlight important data.
  • Surface Charts: Order data categories, use texture coding.
  • Scatterplots: Limit to two dimensions, maintain consistent scale.
  • Bar Graphs: Orient bars consistently, highlight important data.
  • Pie Charts: Use with caution, follow specific guidelines.

General Guidelines:

Keep it simple, clear, and precise. Maximize data density. Use consistent labeling and formatting.

Components in the context of user interface design refer to the various elements that make up the visual and interactive aspects of a software application or system.

Text, messages, icons, colors, buttons, controls, and other visual elements that users interact with to navigate, input information, and receive feedback.

Icons: visual images representing objects and actions for user interaction.

They can stand alone or be grouped in toolbars on desktops or windows. Icons reinforce information, like warning symbols in dialog boxes.

Kinds of Icons

  • Resemblance: Looks like what it means, e.g., a book icon.
  • Symbolic: Abstract representation, e.g., cracked glass for fragility.
  • Exemplar: Illustrates a characteristic, e.g., a knife and fork for a restaurant.
  • Arbitrary: Learned meaning, not directly related.
  • Analogy: Associated physically or semantically, e.g., a wheelbarrow of bricks for the move command.

Characteristics of Icons

  1. Icons have syntax (structure), semantics (meaning), and pragmatics (production).
  1. Syntax: Includes shape, color, and size for classification.
  1. Semantics: Represents meanings like files or wastebaskets.
  1. Pragmatics: Involves physical production, considering screen resolution.
  1. Effectiveness is determined by syntax, semantics, and pragmatics.

Colors: Aiding in communication, organization, and visual appeal.

  • Assist in formatting, grouping, and highlighting.
  • Serve as visual codes for identification.
  • Realistically portray natural objects.
  • Increase screen appeal.

Choosing Colors:

Organize, contrast, sequence, or group information effectively.

Color in Context:

Explain the different types of messages.

Communication provided on the screen to the screen viewer.

System Messages:

Status Messages: Update users on lengthy operations with progress indicators.

Informational Messages: Share general information, often marked with an "I" icon.

Warning Messages: Seek user decisions on potentially problematic actions ("!" icon).

Critical Messages: Demand immediate user action, often marked with "Do Not" or "Stop" signs.

Question Messages: Ask questions and provide choices with a "?" icon.

Instructional Messages:

  • Offer guidance on interacting with the screen.
  • Tailor instructional information to the user's required detail level.
  • Visually distinguish instructional messages from other screen elements.

Foreground colours. - Crucial role in enhancing visibility and emphasizing elements within a UI.

  1. Contrast for Readability: To create high contrast with the background for optimal readability, particularly in text and data.
  1. Text and Data Considerations: Choose black for text on a light-color background with low intensity. For dark backgrounds, use desaturated spectrum colors like white, yellow, or green to enhance visibility.
  1. Emphasis and De-emphasis:

    Highlight important elements with a light value of the foreground color, pure white, or yellow.

    De-emphasize elements by using a dark value of the foreground color to lowlight them.

  1. Warm and Active Colors: Consider warmer and more active foreground colors to add vibrancy and engagement to the interface.

Iterative design and prototyping Design

Methods used in the design process to overcome challenges arising from incomplete requirements. Prototypes, which simulate system features, play a crucial role in this iterative approach. Three main prototyping approaches are:

  1. Throw-away Prototyping:
    • Prototype is built and tested.
    • Design knowledge gained informs the final product.
    • The actual prototype is discarded.
  1. Incremental Prototyping:
    • Final product built in separate components.
    • Overall design partitioned into smaller, independent components.
    • Subsequent releases add one more component each time.
  1. Evolutionary Prototyping:
    • Prototype serves as the basis for the next iteration of design.
    • System evolves from a limited initial version to the final release.

Cognitive models Offering a framework to simulate human problem-solving. They represent the designer's intended mental model for user-system interaction, encompassing ideas about system organization and operation.

  1. Hierarchical Representation: Cognitive models break tasks into steps for a hierarchical approach.
  1. Linguistic and Grammatical Models: Focus on user input alignment with system understanding.
  1. Physical and Device-Level Models: Crucial for designing intuitive user-device interactions.

Goal and Task Hierarchies:

Goals: User goals serve as memory points, aiding evaluation and error correction.

Operators: Basic actions required for system use, from key presses to mental processes.

Methods: Systematic breakdown of goals into subgoals, enhancing task efficiency

Selection: Predictive method selection based on user characteristics and system state.

Example Scenario: Sending a text message

  • Goal: Send a text message.
  • Operators: Unlock phone, open messaging app, type message, send.
  • Methods: Standard method - use fingers for typing, or Voice-to-Text method.
  • Selection Rules: Choose the method based on user preference or accessibility needs.

LINGUISTIC MODELS

Formal systems designed to analyze and describe user-computer interactions using language.

BNF, TAG which help understand user behavior and cognitive aspects in system dialogues.

1. Backus-Naur Form (BNF): Describes dialog grammar at a syntactic level.

Syntax Specification: Commonly used for programming languages and system dialogs.

Example: Graphics system line-drawing function represented using terminals and non-terminals.

Measurements: Evaluated by the number of rules, + and | operators, considering complications like syntax-semantics mismatch.

2. Task–Action Grammar (TAG):

Critique of BNF: BNF measures criticized for lacking cognitive depth and consistency.

TAG Features:

Consistency Emphasis: Aims to enhance language structure and command name consistency.

World Knowledge Encoding: Uses parametrized grammar rules to encode user's world knowledge.

Consistency in TAG: Explicit representation of consistency using UNIX commands as an example.

Other Uses of TAG: Models user's existing knowledge, congruence between features and commands, and includes derived rules.

Cognitive Architectures:
Explores the human mind's structure for intelligent behavior. It utilizes cognitive psychology to create computer-based models, serving as blueprints for artificial intelligence.

Beyond theory, it embodies a hypothesis about universal aspects of human cognition, implemented as executable software.

Models:

  1. LIDA (Learning Intelligent Distribution Agent): Integrates human cognition aspects.
  1. ACT-R (Active Control of Thought-Rational): Simplifies cognitive functions into basic operations.
  1. Soar: Creates generalized intelligent agents for diverse tasks, emulating human cognitive capacity.
